Optimizing Mobile Games for Best Performance

March 28, 2025 8 min read JustCool Tech Team

Creating a visually stunning game that runs smoothly across a wide range of mobile devices is one of the biggest challenges in mobile game development. In this post, we'll share the optimization techniques we used in Boat Attack to achieve consistent 60fps gameplay while maintaining high visual fidelity.

Performance Targets

Before diving into optimization, it's crucial to establish clear performance targets:

  • 60fps on high-end devices (Snapdragon 8 series, A15 Bionic and newer)
  • 30fps on mid-range devices (3-5 years old)
  • Stable framerate with no noticeable hitches
  • Battery efficiency - less than 20% drain per hour
  • Thermal limits - no thermal throttling after 30 minutes of play

Critical Optimization Areas

1. Rendering Pipeline

The rendering pipeline is often the biggest performance bottleneck. Here's how we optimized it:

Pro Tip

Always profile before optimizing. Use Unity's Frame Debugger and Profiler to identify actual bottlenecks rather than guessing.

Technique Performance Gain Visual Impact
Custom SRP (Scriptable Render Pipeline) 25-30% faster Minimal
GPU Instancing for environment 15% faster None
LOD (Level of Detail) System 20% faster Controlled
Occlusion Culling 10-40% faster None

2. Memory Management

Mobile devices have limited RAM, making efficient memory usage critical:

// Example of our object pooling system
public class BoatPool : MonoBehaviour {
  private Queue<GameObject> pool = new Queue<GameObject>();
  public GameObject GetObject() {
    if (pool.Count > 0) {
      return pool.Dequeue();
    }
    return Instantiate(prefab);
  }
  public void ReturnObject(GameObject obj) {
    obj.SetActive(false);
    pool.Enqueue(obj);
  }
}

3. Asset Optimization

Proper asset preparation can make a huge difference:

  • Texture Compression: ASTC for newer devices, ETC2 fallback
  • Mesh Optimization: Average triangle count per boat: 8,000
  • Audio Compression: Vorbis at ~96kbps quality
  • Shader Complexity: Limited to 50 instructions for fragment shaders

Device-Specific Optimizations

We implemented a tiered quality system that automatically adjusts based on device capabilities:

// Device tier detection snippet
DeviceTier GetDeviceTier() {
  if (SystemInfo.graphicsMemorySize >= 4000 &&
    SystemInfo.processorFrequency >= 2500) {
    return DeviceTier.High;
  }
  else if (SystemInfo.graphicsMemorySize >= 2000) {
    return DeviceTier.Medium;
  }
  return DeviceTier.Low;
}

Tools We Used

These tools were invaluable for our optimization process:

  • Unity Profiler: For CPU/GPU/memory profiling
  • RenderDoc: For frame debugging
  • Android GPU Inspector: For low-level GPU analysis
  • Xcode Instruments: For iOS performance analysis
  • Unity Memory Profiler: For tracking memory allocations

Lessons Learned

Our key takeaways from the optimization process:

  • Optimize early and often: Don't leave it until the end
  • Test on real devices: Emulators don't tell the full story
  • Focus on bottlenecks: Optimize what actually matters
  • Maintain quality: Don't sacrifice too much visual fidelity
  • Automate testing: Use automated performance regression tests
